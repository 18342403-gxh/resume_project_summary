## 项目简介

面向新能源车企的智慧出行数字化底座，横跨 C 端车主服务（小程序 / H5 / App）与 B 端运营管理后台，基于一码多端 + 微前端架构实现全域触点覆盖。C 端提供车辆绑定、远程控车、充电服务等车主全生命周期能力，B 端提供白屏化运营配置、多租户管理、数据看板等平台化能力，共同构成车企数字化转型的技术底座，支撑能力的标准化输出与商业化复制。

## 项目技术

- **跨端架构**：Taro 3 + React（C 端一码多端）/ Umi 4 + React（B 端 SPA），统一技术生态降低维护成本
- **微前端**：qiankun 实现 B 端后台模块级拆分，支持独立部署与灰度发布
- **状态管理**：Zustand + React Query，C/B 端统一状态方案，服务端缓存策略复用
- **UI 体系**：C 端 NutUI 二次封装 / B 端 Ant Design Pro，共享设计 Token 保证品牌一致性
- **端侧桥接**：自研 JSBridge SDK（C 端），封装 30+ 原生能力，Promise 化调用
- **权限与多租户**：RBAC 权限模型 + 租户隔离中间件，支持 10+ 车企独立配置
- **工程化**：pnpm Monorepo 统一管理 C/B 端代码，Turborepo 构建编排，共享 @car-platform/* 内部包

## 个人贡献

1. **车辆服务模块（C 端）**：独立主导车辆绑定、远程控车、车况查询等核心功能，覆盖 5 个一级页面与 12 个交互流程，配合后端完成全链路联调与多轮体验优化。
2. **充电业务模块（C 端）**：端到端负责充电桩地图、扫码充电、充电记录等功能，接入高德地图 SDK 并实现聚合渲染方案。
3. **运营配置模块（B 端）**：深度参与后台运营配置中心开发，实现活动页可视化搭建、Push 策略配置、Banner 管理等功能，支撑运营团队白屏化操作。
4. **跨端基础设施**：负责 C/B 端共享工具包（@car-platform/utils）的维护与迭代，沉淀 20+ 通用方法；参与 JSBridge SDK 能力扩展，新增 8 个原生桥接接口。
5. **工程规范与协作**：配合团队推进 Monorepo 改造、ESLint 规则统一、CR 流程规范化，参与 6 个高频业务组件的沉淀与文档编写。
